Why FRAC matters (correction from an earlier draft). The first draft used FRAC-0.5 decoys, which leak (Casanovo reconstructs abundant real peptides from 50%-retained peaks). On the clean FRAC-0.8 decoy three conclusions flip: the two decoy designs agree (not 12× apart); the de novo pre-filter works (171→17k); and mokapot helps a lot (the clean decoy makes Casanovo confidence a real discriminator).

1. The decoy-spectra null

Keep ~20% of a real spectrum's peaks, replace the rest from the global real-fragment pool (precursor preserved); de novo-sequence both real and decoy spectra, BLAST both, compete. Generator scripts/denovo_decoy_gen.py (FRAC 0.8).

4. Scoring: bitscore vs E-value vs mokapot — and the role of peptide length

Raw bitscore is length-confounded (longer peptide → higher bitscore for the same match), so a global bitscore cut under-accepts short peptides. Three fixes, all tested on the EXTRA nr-Eukaryota search at 1% FDR (decoy-spectra null):

scoringpeptides @ 1% FDRnote
bitscore (raw)10,404length-confounded
E-value13,074length-normalized (works under EXTRA — --min-score 1 only disables the E-value filter)
mokapot linear [bitscore + pep-length]14,706additive length term beats E-value
mokapot linear [bitscore + E-value + length + %id]19,556best homology-only score
mokapot GBT [bitscore + length]9,335overfits — worse than raw bitscore
mokapot GBT [bitscore + E-value + length + %id]12,352overfits — worse than linear
Length is handled best by a linear model with E-value, not a fancier one. A linear mokapot using bitscore + E-value + length + %identity recovers 19,556 (+50% over E-value, +88% over raw bitscore). The non-linear gradient-boosted model overfits the decoy in the semi-supervised loop and does worse — which is exactly why Percolator/mokapot default to a linear model. "Use length optimally" = the linear model with the length-normalized E-value, not a deeper model.

The length story, in one table. Raw bitscore is starved of short peptides (only 11 at ≤15 aa, 946 at 16–20 — fewer than even standard, because the bitscore cut penalizes length). E-value recovers them (length-normalized: 60 / 1,769), and mokapot recovers far more (1,661 / 5,488) by combining E-value + length + bitscore. Caveat: those recovered short peptides pass the chance-homology FDR but are the least species-specific (a ≤15-mer matches many organisms), so they rely on the LCA step (§9) for the actual species call, not on the homology hit alone.

The decoy-spectra null is confidence-led; the decoy-database null is homology-led. With the full feature set the top two are longest_run (+2.21, longest high-confidence stretch) and n_hi (+1.79, confident residues that matched the reference) — both confidence-derived; mean_conf collapses to ~0 because the granular longest_run/n_hi/CWI carry its signal. pred_count (+0.70, the 5th-strongest feature) is a reproducibility signal — how many spectra Casanovo de-novo-predicted to the same peptide string across the 9 runs (real max 747 spectra; the clean decoy is almost all one-offs, max 113). A peptide called from hundreds of spectra is more credible than a singleton; the caveat is these are repeated de novo predictions, not database PSMs, so similar spectra can repeat the same error (correlated, not independent evidence) — which is why it helps modestly (+1.9% yield) and sits mid-pack rather than topping the list. The negative bitscore (−1.03) is a collinearity artifact (redundant once E-value/n_hi are in). The decoy-database null instead is led by raw bitscore (+3.27) with confidence demoted (+1.64) — there target and decoy are the same peptide vs different DBs, so confidence cancels. That is why mokapot helps most with the decoy-spectra null: it adds the de novo-confidence dimension the homology-only decoy-database cannot see.

8. Ground-truth check: does the full-feature yield translate to correctness?

The 123,790 is a yield at 1% decoy-spectra FDR — it counts peptides whose real-spectrum-vs-noise separation clears the null. That is not the same as "123,790 verified-correct peptides." To test correctness we need ground truth, so we repeat the exercise on the HeLa entrapment, where a Sage database search (spectrum q≤0.01, ≥7 aa, I/L-normalized) gives an independent "this peptide is really in the sample" label. A de novo peptide is scored correct if its homology hit's query peptide is Sage-confirmed.

FDR scoring (HeLa, nr-hit de novo peptides)accepted @1%Sage-confirmed precisioncorrect recovered
raw bitscore (accept all hits)5,06842.4% (baseline)2,151
mokapot, 8-feature4,09049.4%2,021
mokapot, full 12-feature4,21647.7%2,009
Two honest conclusions. (1) mokapot's accepted set is enriched for genuinely-correct peptides — precision rises from the 42.4% baseline to ~48–49%, so the model concentrates real homology, it does not just inflate counts. (2) But the full 12-feature model does not beat the simpler one on ground-truth correctness: it accepts ~3% more peptides (4,216 vs 4,090) at slightly lower precision (47.7% vs 49.4%) and recovers the same number of correct ones (2,009 vs 2,021). The granular confidence/traceback features (longest_run, n_hi, CWI…) raise yield at fixed decoy-FDR without adding correctness — on a saturated EXTRA decoy they let mokapot exploit the real-vs-noise confidence gap more aggressively, which moves the count but not the truth. So read 123,790 as "peptides that pass the de novo-error + homology FDR," and for a confident species call still layer a confidence/agreement threshold (de novo error drops to 21% at conf≥0.95) and LCA on top — the headline yield is a starting pool, not a verified-correct set.

Caveat on the test: HeLa here is the standard search with the f05 decoy (the only setup where both Sage truth and a trainable null exist); ocelot's 123,790 is EXTRA + clean f08. The transferable lesson is the model comparison (full ≱ simple for correctness) and the precision floor (~48%, a lower bound since Sage truth is itself incomplete), not the absolute HeLa counts.

9. Compute cost: EXTRA is ~5× standard

nr search (312,820 peptides, nr-Euk, 8×32-core array)wall (parallel)total computerelative
Standard (E≤1)~14 min~44 core-hr1×
EXTRA (ultra-sensitive)~69 min~217 core-hr~5×

Standard as a single job on full nr is ~31 min. EXTRA's ~5× cost buys the +short-peptide homology and the confidence-driven mokapot gains.

We BLAST all peptides, then threshold — not the 1% FDR set. The query is all 312,820 Casanovo peptides (only structural filters: ≥7 aa, deduplicated; no confidence/FDR pre-filter), and the 1% FDR is applied to the hits afterward. So compute scales with the full peptide set, which is what makes EXTRA's ~5× sting on large cohorts.

The lever — a de novo pre-filter before BLAST. Keeping only the peptides that pass 1% de novo FDR first (~17k by score, ~31k by confidence, ~63k by mokapot — §5) and BLASTing just those would cut the BLAST workload roughly 5–20×. Trade-off: it discards low-confidence peptides before they are ever tested for homology — the de novo FDR controls "real spectrum vs noise," which is not the same as "has homology" — so it trades recall for speed plus an extra de novo-quality gate. This pre-filter is only viable on a clean FRAC-0.8 decoy (on the leaky f05 it kept just 171 peptides; §2, §5). Default today: BLAST-all; switch to pre-filter when compute (large cohorts × EXTRA) is the bottleneck.

10. What limits a species call — and the recommended pipeline

error moderate (HeLa entrapment)control
chance / spurious homology~0% (clean decoy)decoy-spectra OR reversed-DB FDR (they agree)
de novo sequence error45% raw; 21% at conf≥0.95; 12% at conf≥0.99confidence / mokapot
species mis-assignment36% non-mammal by best-hit (conserved peptides)LCA over all hits
Recommended pipeline. (1) FRAC-0.8 decoy spectra. (2) EXTRA DIAMOND (or standard if compute-bound; EXTRA adds short peptides at ~5× cost). (3) FDR by a linear mokapot on E-value + length + bitscore + Casanovo confidence against the decoy-spectra null — this controls chance homology AND de novo error in one model, and the clean decoy makes the confidence term the strongest signal. Calibrate on E-value, never raw bitscore. (4) LCA over all hits for species. The reversed-database FDR is a valid, equivalent check on chance homology alone.
